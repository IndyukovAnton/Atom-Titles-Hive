# Архитектура рефакторинга HomePage

## Обзор
Рефакторинг `HomePage.tsx` направлен на разделение ответственности и улучшение читаемости кода.
Монолитный компонент разбит на:
- **Умные хуки**, инкапсулирующие бизнес-логику и работу с API.
- **Глупые (презентационные) компоненты**, отвечающие за отображение.

## Структура

### Хуки (`src/hooks/`)
1.  **`useMediaData`**:
    *   Отвечает за загрузку списка медиа (`MediaEntry[]`).
    *   Управляет состоянием загрузки и ошибок медиа.
    *   Реагирует на изменение `selectedGroupId`.

2.  **`useGroupManagement`**:
    *   Отвечает за загрузку статистики групп (`GroupStats`).
    *   Управляет операциями удаления и обновления списка групп.
    *   Управляет состоянием модальных окон создания/редактирования группы.

3.  **`useDragAndDrop`**:
    *   Содержит логику обработки завершения перетаскивания (`onDragEnd`).
    *   Обновляет локальное состояние порядка групп (optimistic UI).

### Компоненты (`src/components/HomePage/`)
1.  **`HomeHeader`**:
    *   Верхняя панель с заголовком, кнопкой добавления и меню пользователя.
    *   Получает колбэки для навигации и действий.

2.  **`Sidebar`**:
    *   Боковая панель со списком групп.
    *   Содержит `DragDropContext` и `Droppable` зону.
    *   Использует `SidebarGroupItem`.

3.  **`SidebarGroupItem`**:
    *   Отдельный элемент группы в сайдбаре.
    *   Является `Draggable`.
    *   Содержит контекстное меню (Редактировать, Удалить).

4.  **`MediaGrid`**:
    *   Отображает сетку медиа-карточек или сообщения о пустом списке/ошибке.
    *   Использует `MediaCard`.

5.  **`MediaCard`**:
    *   Карточка отдельной медиа-записи.
    *   Отображает изображение, рейтинг, категорию.
    *   Мемоизирован (`React.memo`) для производительности.

## Схема данных
`HomePage` является корневым контейнером, который связывает хуки и компоненты:
[HomePage]
   |-- useMediaData --> { mediaList } --> [MediaGrid]
   |-- useGroupManagement --> { groupStats } --> [Sidebar]
   |-- useDragAndDrop
