# Решение по стилизации: TailwindCSS + shadcn/ui

## Контекст
Проект находится в стадии активной разработки, но имеет накопленный технический долг в виде смешанных подходов к стилизации (Tailwind, Vanilla CSS, инлайн-стили). Это привело к сложностям в поддержке и неконсистентному UI.

## Решение
Для унификации стилей и ускорения разработки выбран стек:
- **TailwindCSS (v4)**: Utility-first фреймворк для верстки. Обеспечивает высокую производительность и гибкость.
- **shadcn/ui**: Набор переиспользуемых компонентов, построенных на Radix UI и Tailwind CSS. Обеспечивает доступность и современный визуальный стиль.
- **Lucide React**: Библиотека иконок.

## Почему не CSS Modules?
Хотя CSS Modules решают проблему изоляции стилей, они требуют написания кастомного CSS, что замедляет разработку по сравнению с использованием готовых дизайн-токенов и утилит Tailwind.

## План действий
1. Настроить TailwindCSS v4 (уже выполнено, используется `@tailwindcss/vite`).
2. Установить базовые компоненты shadcn/ui.
3. Мигрировать существующие страницы и компоненты.
4. Удалить легаси CSS файлы.

## Стандарты кода
- Использовать утилиту `cn()` для объединения классов.
- Избегать инлайн-стилей (`style={{...}}`).
- Все цвета и отступы брать из дизайн-системы (Tailwind config).
- Поддерживать темную/светлую тему через CSS-переменные (реализовано в shadcn).
