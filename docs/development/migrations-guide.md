# Руководство по миграциям базы данных

В этом проекте для управления схемой базы данных используется TypeORM Migrations.

## Основные команды

Все команды выполняются из директории `backend`.

### 1. Генерация миграции
Используется при изменении `entity` классов. TypeORM сравнивает текущие сущности с состоянием базы данных и генерирует SQL для обновления.

```bash
npm run migration:generate src/migrations/NameOfMigration
```
*Пример: `npm run migration:generate src/migrations/AddUserRole`*

### 2. Применение миграций
Применяет все незавершенные миграции к базе данных.

```bash
npm run migration:run
```

### 3. Откат миграций
Откатывает последнюю примененную миграцию.

```bash
npm run migration:revert
```

### 4. Просмотр статуса
Показывает список миграций и их статус (применена/не применена).

```bash
npm run migration:show
```

## Работа в Production

В production окружении опция `synchronize` отключена. Все изменения схемы БД должны проходить через миграции.

### Workflow для разработчика:
1. Внесите изменения в `.entity.ts` файлы.
2. Запустите `npm run migration:generate ...` с понятным названием.
3. Проверьте созданный файл в `src/migrations`.
4. Запустите `npm run migration:run` для проверки на локальной БД.
5. Закоммитьте файл миграции.

## Troubleshooting

Если возникли проблемы с рассинхронизацией (например, вы удалили файл миграции, но она помечена как выполненная в БД):
1. Откройте БД (например, через DBeaver).
2. Найдите таблицу `migrations`.
3. Удалите запись о проблемной миграции.
