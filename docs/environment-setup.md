# Настройка окружения

## Обзор
Приложение использует переменные окружения для конфигурации поведения в разных средах (Local, Development, Staging, Production).

## Файлы конфигурации
- `.env.local`: Локальная конфигурация (не коммитится в Git)
- `.env.development`: Конфигурация для development окружения
- `.env.production`: Конфигурация для production окружения
- `.env.example`: Пример конфигурации (коммитится в Git)

## Переменные окружения

| Переменная | Описание | Обязательна | Пример |
|------------|----------|-------------|--------|
| `VITE_API_URL` | URL API бэкенда | Да | `http://localhost:3553` |
| `VITE_API_TIMEOUT` | Таймаут запросов в мс | Да | `30000` |
| `VITE_APP_NAME` | Название приложения | Нет | `Atom Titles-Hive` |

## Использование в коде
Конфигурация доступна через модуль `src/config`:

```typescript
import { config } from '@/config';

console.log(config.apiUrl);
```

## Troubleshooting

### API URL не обновляется
Если вы изменили `.env` файл, перезапустите dev-сервер (`npm run dev`), так как Vite не всегда подхватывает изменения переменных окружения "на лету" без перезагрузки.

### Ошибка "Environment variable ... is missing"
Убедитесь, что вы создали файл `.env.local` или соответствующий `.env` файл для вашего режима, и он содержит все обязательные переменные. Проверьте `.env.example` для справки.

### CORS ошибки
Убедитесь, что `VITE_API_URL` указывает на правильный адрес бэкенда. Если используется прокси, проверьте настройки в `vite.config.ts`.
