# Задача AUTH-01: Интеграция авторизации через Google

## Зависимости
- BE-01: Базовая система авторизации (JWT)
- FE-01: Существующие страницы Login/Register

## Описание проблемы
В приложении доступна только стандартная авторизация по логину/паролю. Для улучшения пользовательского опыта необходимо добавить возможность быстрого входа через Google.

## Цель задачи
Реализовать полноценный цикл OAuth2 авторизации через Google (Backend + Frontend).

## Чек-лист выполнения

### Этап 1: Backend (NestJS)
- [x] Установить зависимости: `passport-google-oauth20`, `@types/passport-google-oauth20`.
- [x] Создать `GoogleStrategy` в модуле авторизации.
- [x] Добавить в `.env` переменные: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_CALLBACK_URL`.
- [x] Реализовать эндпоинты в `AuthController`:
  - `GET /auth/google` — перенаправление на Google.
  - `GET /auth/google/callback` — обработка ответа от Google, создание/поиск пользователя, выдача JWT.
- [x] Реализовать логику автоматической регистрации пользователя при первом входе через Google.

### Этап 2: Frontend (React)
- [x] Добавить кнопку "Войти через Google" на страницы Login и Register.
- [x] Стилизовать кнопку в соответствии с брендом Google и общим дизайном приложения.
- [x] Реализовать редирект на backend эндпоинт при нажатии.
- [x] Обработать успешный возврат: получение токена и сохранение в AuthContext/LocalStorage.
- [x] Добавить уведомление (Toast) об успешном входе или ошибке.

## Критерии приёмки
- ✅ Пользователь может войти в систему, используя Google account.
- ✅ Если пользователя не существует, аккаунт создается автоматически (email берется из Google).
- ✅ Существующие пользователи могут "привязать" Google, если email совпадает.
- ✅ Кнопка Google Auth выглядит нативно и красиво.
