# Задача BE-02: Реализация системы логирования

## Зависимости
- Нет зависимостей

## Описание проблемы
Согласно project.md и README.md, логирование должно вестись в папку`/logs` с записью в файлы формата `YYYY-MM-DD.txt`. На данный момент эта функциональность не реализована.

Требования из документации:
- Логи записываются в `/logs/YYYY-MM-DD.txt`
- Формат записи: `[HH:MM:SS] [LEVEL] Сообщение`
- Уровни: INFO, WARN, ERROR

## Цель задачи
Реализовать кастомный сервис логирования для записи в файлы, интегрировать его во все модули backend.

## Реализовано

### Компоненты
1. **LoggerService** (`src/utils/logger.service.ts`)
   - Методы: `log()`, `warn()`, `error()`
   - Автоматическое создание папки `logs/`
   - Форматирование логов: `[HH:MM:SS] [LEVEL] Message`
   - Метод `cleanOldLogs()` для ротации

2. **HttpLoggerMiddleware** (`src/utils/http-logger.middleware.ts`)
   - Логирование входящих запросов
   - Логирование времени выполнения
   - Логирование HTTP ошибок (4xx, 5xx)

3. **AllExceptionsFilter** (`src/filters/all-exceptions.filter.ts`)
   - Перехват необработанных исключений
   - Логирование с stack trace
   - Форматирование ответов об ошибках
