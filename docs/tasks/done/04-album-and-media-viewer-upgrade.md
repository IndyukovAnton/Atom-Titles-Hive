# Задача FE-04: Модернизация Системы Альбомов и Просмотра Фото

## Зависимости
- FE-04: `MediaDetailPage.tsx`, `Album` компонент

## Описание проблемы
Альбом реализован неудобно: при добавлении фото происходит редирект, альбом вынесен отдельно, нет системы сортировки и удобного просмотра во весь экран.

## Цель задачи
Объединить альбом с основной страницей записи, внедрить сеточную систему добавления фото и продвинутый Telegram-like просмотрщик.

## Чек-лист выполнения

### Этап 1: Интеграция Альбома
- [ ] Перенести функционал альбома непосредственно на страницу `MediaDetailPage` (расположить ниже основной информации).
- [ ] Убрать лишние редиректы при добавлении медиа в альбом.

### Этап 2: Система "Ячеек" (Photo Grid)
- [ ] Реализовать интерфейс добавления через пустые ячейки (слоты): нажал -> выбрал фото -> оно заняло ячейку.
- [ ] Автоматическое добавление новой пустой ячейки при заполнении текущих.
- [ ] Внедрить Drag & Drop для изменения порядка фотографий в альбоме.
- [ ] Снять лимит на количество изображений.

### Этап 3: Telegram-style Photo Viewer
- [ ] Создать (или интегрировать) компонент полноэкранного просмотра:
  - Центрирование и перемещение курсором (pan & zoom).
  - Приближение/отдаление (скролл или жесты).
  - Листание (swipe/arrows) на соседние фото из альбома.
- [ ] **Багфикс**: Исправить ошибку `DialogContent requires a DialogTitle` (добавить скрытый заголовок для доступности).

## Критерии приёмки
- ✅ Альбом находится на той же странице, что и запись.
- ✅ Фотографии перетаскиваются (меняется очередь отображения).
- ✅ Интерфейс добавления фото через ячейки интуитивен и быстр.
- ✅ Просмотрщик фото работает плавно, поддерживает зум и навигацию между изображениями.
- ✅ Консоль чиста от ошибок Radix UI при открытии модалок.

## Тестирование
- [ ] Загрузить 5+ фотографий в альбом.
- [ ] Изменить их порядок перетаскиванием и проверить сохранение.
- [ ] Открыть фото, увеличить масштаб, подвигать и переключиться на следующее.
- [ ] Проверить адаптивность сетки ячеек на разных экранах.
