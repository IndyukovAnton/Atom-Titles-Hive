# Задача FE-05: Визуальная полировка и UI Fixes

## Зависимости
- FE-05: `NotificationCenter`, `LoadingScreen`, `AuthForms`, `MediaItem`

## Описание проблемы
Ряд мелких визуальных багов и недочетов UX портят общее впечатление от "Premium" продукта.

## Цель задачи
Исправить визуальные несоответствия, настроить систему уведомлений и синхронизировать загрузку.

## Чек-лист выполнения

### Этап 1: Уведомления (Toasts)
- [ ] Переместить уведомления в правую часть экрана.
- [ ] Добавить в каждый Toast кнопку закрытия (крестик).
- [ ] Визуализировать время жизни: добавить убывающую полосу прогресса (progress bar).
- [ ] Уменьшить стандартное время отображения уведомлений.

### Этап 2: Загрузка и Onboarding
- [ ] Исправить порядок отображения: Обучение (Onboarding) должно инициализироваться и показываться **только после** полного исчезновения загрузочного экрана (Loading Screen).

### Этап 3: Формы и Элементы
- [ ] **Password Icon**: Исправить цвета иконки глаза. В обычном состоянии — серый, при наведении — черный (не белый, чтобы не сливаться с фоном).
- [ ] **Media Records**: Исправить "уродское растяжение" карточек записей. Сделать их аккуратными, одинакового размера или с правильным сохранением пропорций (aspect-ratio).

## Критерии приёмки
- ✅ Уведомления не перекрывают важные части интерфейса и управляются пользователем.
- ✅ Обучение не "выскакивает" раньше времени.
- ✅ Иконки в формах контрастны и правильно реагируют на hover.
- ✅ Сетка записей выглядит чисто и аккуратно.

## Тестирование
- [ ] Вызвать несколько уведомлений подряд.
- [ ] Перезагрузить страницу и проверить момент появления подсказок обучения.
- [ ] Проверить все состояния поля пароля.
- [ ] Проверить отображение записей на мобильных и десктопных разрешениях.
