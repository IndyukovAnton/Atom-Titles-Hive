# Задача FE-07: Оптимизация и рефакторинг компонентов (UI vs Logic)

## Описание проблемы
В проекте присутствуют крупные компоненты (например, `AddMediaModal`, `NotificationCenter`), в которых переметана бизнес-логика, работа с API и верстка. Это затрудняет тестирование, переиспользование UI-элементов и приводит к избыточным перерендерам.

## Цель задачи
Разделить компоненты на "умные" (логические/контейнеры) и "глупые" (презентационные/UI), вынести общую логику в кастомные хуки и расширить библиотеку базовых UI-компонентов.

## Чек-лист выполнения

### Этап 1: Анализ и выделение UI-атомов
- [ ] Проанализировать крупные компоненты (`AddMediaModal`, `NotificationCenter`, `Sidebar`) на наличие повторяющихся UI-паттернов.
- [ ] Вынести общие элементы в `src/components/ui`:
  - Кастомные кнопки, инпуты (с учетом стилей проекта).
  - Элементы списков, карточки.
  - Контейнеры для модалок и выпадающих меню.
- [ ] Обеспечить, чтобы UI-компоненты принимали данные через props и не зависели от глобального стейта или API напрямую.

### Этап 2: Вынос логики (Hooks & Containers Pattern)
- [ ] Для сложных форм и модулей создать кастомные хуки (например, `useMediaForm`, `useNotifications`, `useTreeManagement`).
- [ ] Перенести в хуки:
  - Валидацию данных.
  - Вызовы API.
  - Обработку локального состояния.
- [ ] Оставить в основных компонентах только композицию UI-элементов и передачу данных из хуков.

### Этап 3: Оптимизация производительности
- [ ] Использовать `React.memo` для предотвращения лишних ререндеров чисто презентационных компонентов.
- [ ] Оптимизировать тяжелые вычисления с помощью `useMemo` и коллбэки с помощью `useCallback`.
- [ ] Убедиться, что контексты (Context API) не вызывают ререндер всего приложения при изменении локальных данных.

## Критерии приёмки
- ✅ Код компонентов стал чище и короче (логика отделена от верстки).
- ✅ UI-элементы легко переисполняются в разных частях приложения.
- ✅ Повысилась производительность (меньше лишних перерендеров).
- ✅ Базовые UI-компоненты задокументированы или имеют понятные интерфейсы (TypeScript).

## Тестирование
- [ ] Проверить работоспособность всех отрефакторенных модулей.
- [ ] Убедиться, что функциональность не изменилась (регрессионное тестирование).
- [ ] С помощью React DevTools убедиться в отсутствии избыточных ререндеров.
