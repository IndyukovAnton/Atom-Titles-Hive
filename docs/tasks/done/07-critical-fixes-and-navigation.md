# Задача 07: Критические исправления и навигация (MVP)

## Описание
Первоочередная задача по стабилизации базового функционала и внедрению улучшенной навигации. Включает исправление известных багов (High Priority) и реализацию системы папок.

## Цель задачи
Обеспечить корректную работу базовых сценариев пользователя и предоставить удобный инструмент структурирования контента через папки.

## Чек-лист выполнения

### Этап 1: Исправление критических багов (High Priority)
- [x] **Страница записи**: Исправить баг с открытием страницы подробной информации при клике на запись.
- [x] **UI Карточек**: Оптимизировать отображение карточек (исправить некорректное растягивание по высоте).
- [x] **Навигация**: Добавить кнопку "Назад" на странице "Профиль".
- [x] **Темизация**: Исправить работу переключателя темы (Light/Dark mode).

### Этап 2: Система папок (Tree Component)
- [x] **Компонент**: Интегрировать компонент `Tree` из библиотеки `@reui`.
- [x] **Структура**: Реализовать поддержку вложенности папок.
- [x] **Управление**: Добавить возможности создания, переименования и удаления папок.
- [x] **Drag-and-Drop**: Реализовать перетаскивание записей между папками.

## Критерии приёмки
- ✅ Переход к подробностям записи работает без ошибок.
- ✅ Карточки имеют одинаковую высоту в ряду или корректный Masonry-layout.
- ✅ Из профиля можно вернуться на предыдущий экран.
- ✅ Тема переключается корректно и сохраняется при перезагрузке.
- ✅ Можно создать многоуровневую структуру папок.
- ✅ Записи перемещаются между папками через Drag-and-Drop.

## Технические требования
- Использовать `@reui` для дерева.
- Состояние папок должно сохраняться (Local Storage или Backend).

## ⚠️ ВАЖНО: Финальная проверка
1. ✅ Баги не воспроизводятся.
2. ✅ Папки работают интуитивно понятно.
3. ✅ Нет регрессий в существующем функционале.
