# Задача 09: Умный поиск и фильтрация

## Описание
Внедрение продвинутой системы поиска и фильтрации для работы с большой базой тайтлов.

## Цель задачи
Обеспечить мгновенный доступ к нужному контенту через оптимизированный поиск и гибкие фильтры.

## Чек-лист выполнения

### Этап 1: Реализация поиска
- [x] **Debounce**: Внедрить `useDebounce` (или аналог) для оптимизации запросов при вводе.
- [x] **Логика поиска**: Реализовать поиск по:
  - [x] Названию.
  - [x] Описанию.
  - [x] Жанрам.
  - [x] Тегам.
- [x] **Автодополнение**: Реализовать выпадающий список подсказок (Autocomplete) при вводе.

### Этап 2: Расширенная фильтрация
- [x] **UI Фильтров**: Создать панель фильтрации.
- [x] **Критерии**:
  - [x] По категориям (Movie, Book, etc.).
  - [x] По датам (Date ranges).
  - [x] По рейтингу (Min/Max).
  - [x] По статусу (Plan to watch, Completed, etc.).

## Критерии приёмки
- ✅ Поиск не вызывает лишних API запросов (Debounce работает).
- ✅ Поиск находит записи по частичному совпадению в разных полях.
- ✅ Фильтры корректно комбинируются между собой.
- ✅ Производительность поиска соответствует требованиям (<300мс реакция).

## Технические требования
- Оптимизация ре-рендеров списка результатов.

## Реализация

### Созданные компоненты и хуки:
1. **useDebounce** - хук для оптимизации поисковых запросов с задержкой 300мс
2. **useSearch** - хук для управления поиском и автодополнением
3. **useFilters** - хук для управления фильтрами медиа
4. **SearchBar** - компонент поиска с автодополнением и клавиатурной навигацией
5. **FilterPanel** - выдвижная панель фильтрации с поддержкой всех критериев
6. **useMediaData** (обновлён) - добавлена поддержка поиска и фильтрации

### Особенности реализации:
- Debounce с задержкой 300мс для оптимизации запросов
- Автодополнение с превью результатов (изображение, описание, рейтинг)
- Клавиатурная навигация (стрелки, Enter, Escape)
- Комбинирование фильтров по категориям, рейтингу, датам, жанрам и тегам
- Отображение активных фильтров в виде badge
- Современный UI с использованием shadcn компонентов
- Адаптивный дизайн для всех экранов
- Отсутствие лишних ре-рендеров благодаря useMemo и useCallback
