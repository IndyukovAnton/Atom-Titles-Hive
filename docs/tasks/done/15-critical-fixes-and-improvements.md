# Критические исправления и улучшения UX

Этот документ содержит список задач, направленных на исправление критических ошибок в скомпилированной версии приложения, а также на значительное улучшение пользовательского интерфейса и систем обучения.

## 1. 🐛 Исправление критических ошибок (Сборка и API)

- [x] **Диагностика авторизации в продакшн-сборке**
  - [x] Выяснить, почему не работает вход/регистрация в сбилженной (built) версии.
  - [x] Проверить конфигурацию CORS и API_URL в продакшн-режиме.
  - [x] Проверить логи сервера при попытке авторизации из статики.
- [x] **Исправление создания Групп**
  - [x] Исправить ошибку при создании группы (отсутствует `id` в ответе или некорректная обработка на фронтенде).
  - [x] Убедиться, что дерево групп обновляется корректно после создания.
- [x] **Исправление создания Записей (Entries)**
  - [x] Диагностика и исправление ошибки "Запись не создаётся".
  - [x] Проверить валидацию данных перед отправкой.

## 2. 🎨 Визуал и Анимации

- [x] **Анимация фона (Auth/Register)**
  - [x] Добавить динамичный и красивый анимированный фон для страниц входа и регистрации.
  - [x] Использовать современные CSS-эффекты или легкие JS-библиотеки (tsparticles, canvas и т.д.).
- [x] **Улучшение эстетики (Главная и Auth)**
  - [x] Провести ревизию дизайна страниц авторизации/регистрации (сделать "premium" вид).
  - [x] Добавить микро-взаимодействия (ховеры, фокус, переходы) на Главную страницу.
  - [x] Убедиться в использовании современной типографики и цветовой палитры (glassmorphism/gradients).

## 3. 🎓 Интерактивное обучение (Onboarding)

- [x] **Внедрение системы Guided Tour**
  - [x] Реализовать пошаговое обучение ("водитель за ручку").
  - [x] Подсветка ключевых элементов интерфейса (Кнопка "Добавить", Дерево групп, Поиск).
  - [x] Всплывающие подсказки с объяснением действий.
  - [x] Возможность пропустить или перезапустить обучение.
  - [x] Использовать библиотеку типа `driver.js` или самописное решение на базе `@radix-ui/react-popover` + overlay.

---

### Примечания
- Приоритет: Сначала багфиксы (п.1), затем визуал и обучение.
- Убедиться, что исправления работают как в `dev` режиме, так и в `preview` (build).
