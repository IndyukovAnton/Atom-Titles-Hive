# Задача REL-00A: Smoke test matrix для MSI (Windows)

## Приоритет
🔴 **P0 / Блокер релиза**

## Цель
Иметь короткий, повторяемый список сценариев, который гарантирует: **MSI установлен → приложение запустилось → данные сохраняются → критичные фичи работают**.

## Окружение
- Windows 10/11 “чистая” (без node/npm/rust)
- Новый пользователь Windows (чистый app data dir)
- Интернет: on/off (оба варианта)

## Матрица тестов (обязательные)

### 1) Установка / Запуск
- [ ] MSI устанавливается без ошибок.
- [ ] Приложение стартует (окно появляется < 5 секунд).
- [ ] Нет “white screen” / бесконечной загрузки.

### 2) Sidecar backend
- [ ] Backend стартует и становится доступен (frontend получает `backend-ready`).
- [ ] При ошибке backend — в UI показывается понятное сообщение (не краш).

### 3) БД и миграции
- [ ] При пустой папке данных БД создаётся автоматически.
- [ ] Миграции применяются автоматически.
- [ ] После закрытия/старта данные остаются.

### 4) Auth
- [ ] Регистрация нового пользователя.
- [ ] Логин существующего пользователя.
- [ ] Логаут.

### 5) CRUD и медиа
- [ ] Создать группу.
- [ ] Создать запись (media entry).
- [ ] Переместить запись (DnD/смена группы).
- [ ] Загрузить файл (media file) и убедиться, что он открывается.
- [ ] Удалить запись и убедиться, что связанные файлы/ссылки не ломают UI.

### 6) Перезапуск / Обновление
- [ ] Перезапуск приложения сохраняет данные.
- [ ] Установка новой версии MSI поверх (upgrade) не ломает данные/схему.

## Артефакты при падении (собирать всегда)
- [ ] Логи Tauri (app data dir)
- [ ] Логи backend (app data dir)
- [ ] Версия приложения / build hash (если есть)
- [ ] Скрин/видео воспроизведения
