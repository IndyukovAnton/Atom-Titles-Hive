# Задача FE-08: Frontend observability (логирование без console.log + no-secrets)

## Приоритет
🟡 **P1 (качество/поддержка)** — не блокирует MSI напрямую, но сильно помогает диагностике.

## Проблема
- В коде встречаются `console.*` и ручные логи ошибок.
- В релизе нельзя логировать чувствительные данные (токены, ключи, пути).
- В desktop режиме важно быстро понять: backend недоступен? порт не получен? запросы падают?

## Цель
Сделать лёгкий слой логирования:
- уровни (debug/info/warn/error),
- выключение debug в production,
- запрет логирования секретов,
- единый подход в `api/client.ts` и ключевых местах.

## Чек‑лист
- [ ] Ввести `frontend/src/utils/logger.ts` (или аналог) и заменить `console.*` в коде на него.
- [ ] В `frontend/src/api/client.ts`:
  - [ ] логировать только метаданные (method/url/status), без токенов и payload с секретами,
  - [ ] корректно логировать сетевые ошибки/401.
- [ ] Добавить ESLint правило `no-console` (разрешить только `console.error` при необходимости).

## Критерии приёмки
- ✅ В production нет debug‑логов
- ✅ Нет логов с токенами/ключами
- ✅ Диагностика проблем sidecar/backend стала быстрее (по логам понятно “что сломалось”)
