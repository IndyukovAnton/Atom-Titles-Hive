# Задача BE-03: Реализация модуля статистики

## Зависимости
- Нет зависимостей

## Описание проблемы
Согласно README.md и requirements, ProfilePage должен отображать подробную статистику:
- Общее количество записей
- Любимая категория (наиболее частая)
- Любимый жанр
- Общее время просмотра
- Время просмотра за текущий месяц

На данный момент `profile.service.ts` возвращает только базовую информацию о пользователе без статистики.

## Цель задачи
Расширить ProfileService для вычисления и возврата подробной статистики пользователя.

## Чек-лист выполнения

### Этап 1: Обновление DTO
- [ ] Создать `backend/src/dto/profile-stats.dto.ts`
  ```typescript
  {
    totalEntries: number;
    favoriteCategory: string | null;
    favoriteGenre: string | null;
    totalWatchTime: number; // в часах
    currentMonthWatchTime: number; // в часах
  }
  ```
- [ ] Обновить `ProfileResponseDto` для включения stats

### Этап 2: Реализация вычисления количества записей
- [ ] В `profile.service.ts` добавить метод `getTotalEntries(userId: number)`
- [ ] Использовать TypeORM count для подсчёта записей пользователя

### Этап 3: Реализация любимой категории
- [ ] Добавить метод `getFavoriteCategory(userId: number)`
- [ ] Группировать записи по category
- [ ] Вернуть наиболее частую категорию
- [ ] Обработать случай отсутствия записей (вернуть null)

### Этап 4: Реализация любимого жанра
- [ ] Добавить метод `getFavoriteGenre(userId: number)`
- [ ] Распарсить JSON массивы genres из всех записей
- [ ] Подсчитать частоту каждого жанра
- [ ] Вернуть наиболее частый жанр
- [ ] Обработать случай отсутствия записей (вернуть null)

### Этап 5: Реализация общего времени просмотра
- [ ] Добавить метод `getTotalWatchTime(userId: number)`
- [ ] Для каждой записи с startDate и endDate вычислить разницу
- [ ] Суммировать все разницы
- [ ] Вернуть результат в часах (округлённо)
- [ ] Обработать записи без дат (пропускать)

### Этап 6: Реализация времени за текущий месяц
- [ ] Добавить метод `getCurrentMonthWatchTime(userId: number)`
- [ ] Фильтровать записи, где endDate в текущем месяце
- [ ] Вычислить сумму времени аналогично общему
- [ ] Вернуть результат в часах

### Этап 7: Интеграция в ProfileController
- [ ] Обновить `getProfile()` для возврата статистики
- [ ] ИЛИ создать отдельный endpoint `GET /profile/stats`

### Этап 8: Оптимизация производительности

- [ ] Рассмотреть кэширование статистики (опционально)

### Этап 9: Документация

- [ ] Создать примеры ответов в документации

## Тестирование

### Подсчёт записей
- [ ] Для пользователя с 5 записями возвращается 5
- [ ] Для пользователя с 0 записей возвращается 0  
- [ ] Записи других пользователей не учитываются

### Любимая категория
- [ ] Корректно определяется наиболее частая категория
- [ ] При равных количествах возвращается любая
- [ ] При отсутствии записей возвращается null
- [ ] Регистр категорий учитывается корректно

### Любимый жанр
- [ ] Корректно парсятся JSON массивы genres
- [ ] Подсчитывается частота каждого жанра
- [ ] Возвращается наиболее частый жанр
- [ ] При отсутствии жанров возвращается null
- [ ] Обрабатываются пустые или некорректные массивы genres

### Общее время просмотра
- [ ] Корректно вычисляется разница между startDate и endDate
- [ ] Суммируется время всех записей
- [ ] Возвращается в часах (не днях, не минутах)
- [ ] Записи без дат пропускаются
- [ ] Результат округляется до разумной точности (например, 1 десятичный знак)

### Время текущего месяца
- [ ] Фильтруются только записи текущего месяца
- [ ] Корректно определяется текущий месяц и год
- [ ] Вычисление аналогично общему времени
- [ ] Для записей не из текущего месяца возвращается 0

### API интеграция
- [ ] `GET /api/profile` (или `/profile/stats`) возвращает все показатели
- [ ] Response соответствует DTO
- [ ] Авторизация работает (требуется JWT)
- [ ] Статус 200 при успехе

### Производительность
- [ ] Запрос выполняется < 500ms для 100 записей
- [ ] Запрос выполняется < 1s для 1000 записей
- [ ] Нет N+1 проблем
- [ ] Используется минимум запросов к БД

### Проверка кода
- [ ] ESLint не выдаёт ошибок
- [ ] TypeScript компилируется без ошибок
- [ ] Все методы типизированы
- [ ] JSDoc комментарии добавлены
- [ ] `npm run build` проходит успешно
