# Задача BE-04: Написание Unit и Integration тестов

## Зависимости
- Нет зависимостей (можно выполнять параллельно с другими)

## Описание проблемы
В backend отсутствуют тесты (кроме заглушки `app.controller.spec.ts`). Для надёжности и поддерживаемости кода необходимо покрытие тестами:
- Unit тесты для services
- Integration тесты для controllers
- E2E тесты для критичных user flows

## Цель задачи
Создать полное покрытие тестами для всех модулей backend (auth, media, groups, profile).

## Чек-лист выполнения

### Этап 1: Настройка тестового окружения
- [ ] Настроить Jest конфигурацию
- [ ] Создать `backend/test/test-setup.ts` для глобальных моков
- [ ] Настроить in-memory SQLite БД для тестов
- [ ] Создать test fixtures и моки данных

### Этап 2: Unit тесты для AuthService
- [ ] Создать `auth.service.spec.ts`
- [ ] Тесты для `register()`:
  - Успешная регистрация
  - Дублирование email/username
  - Валидация данных
  - Хеширование пароля
- [ ] Тесты для `login()`:
  - Успешный логин
  - Неправильный пароль
  - Несуществующий пользователь
  - Генерация JWT токена
- [ ] Покрытие >= 80%

### Этап 3: Integration тесты для AuthController
- [ ] Создать `auth.controller.spec.ts`
- [ ] Тесты для `POST /auth/register`:
  - Status 201 при успехе
  - Status 400 при невалидных данных
  - Status 409 при дублировании
- [ ] Тесты для `POST /auth/login`:
  - Status 200 при успехе + возврат токена
  - Status 401 при неправильных credentials
- [ ] Покрытие >= 80%

### Этап 4: Unit тесты для MediaService
- [ ] Создать `media.service.spec.ts`
- [ ] Тесты для `createMedia()`:
  - Успешное создание
  - Валидация обязательных полей
  - Привязка к пользователю
  - Привязка к группе
- [ ] Тесты для `getMediaByUser()`:
  - Возврат всех media пользователя
  - Фильтрация по группе
  - Пустой результат для нового пользователя
- [ ] Тесты для `updateMedia()` и `deleteMedia()`
- [ ] Покрытие >= 80%

### Этап 5: Integration тесты для MediaController
- [ ] Создать `media.controller.spec.ts`
- [ ] Тесты с авторизацией (mock JWT Guard)
- [ ] CRUD операции (GET, POST, PUT, DELETE)
- [ ] Проверка прав доступа (нельзя редактировать чужие media)
- [ ] Покрытие >= 80%

### Этап 6: Unit тесты для GroupsService
- [ ] Создать `groups.service.spec.ts`
- [ ] Тесты для `createGroup()`, `updateGroup()`, `deleteGroup()`
- [ ] Тест перемещения media при удалении группы
- [ ] Покрытие >= 80%

### Этап 7: Integration тесты для GroupsController
- [ ] Создать `groups.controller.spec.ts`
- [ ] CRUD операции для групп
- [ ] Проверка прав доступа
- [ ] Покрытие >= 80%

### Этап 8: Unit тесты для ProfileService
- [ ] Создать `profile.service.spec.ts`
- [ ] Тесты для `getProfile()`
- [ ] Тесты для статистики (если реализована в BE-03)
- [ ] Покрытие >= 80%

### Этап 9: E2E тесты (критичные flows)
- [ ] Создать `test/auth.e2e-spec.ts`
  - Полный flow: регистрация → логин → получение токена
- [ ] Создать `test/media.e2e-spec.ts`
  - Создание media → получение списка → обновление → удаление
- [ ] Создать `test/groups.e2e-spec.ts`
  - Создание группы → добавление media → удаление группы

### Этап 10: CI Integration
- [ ] Добавить в `package.json` скрипт `test:ci`
- [ ] Настроить генерацию coverage отчёта
- [ ] Документировать как запускать тесты
