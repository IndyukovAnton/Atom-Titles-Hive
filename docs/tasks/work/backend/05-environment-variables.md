# Задача BE-05: Вынос секретов в переменные окружения

## Зависимости
- Нет зависимостей

## Описание проблемы
В `backend/src/modules/auth/auth.module.ts` секрет JWT захардкожен:
```typescript
secret: configService.get('JWT_SECRET') || 'super-secret-key-change-in-production'
```

Это создаёт уязвимость безопасности в production. Все секреты должны быть в `.env` файле.

## Цель задачи
Вынести все чувствительные данные в переменные окружения и обеспечить их обязательную проверку.

## Чек-лист выполнения

### Этап 1: Обновление .env файлов
- [ ] Обновить `backend/.env` с обязательными переменными:
  ```
  JWT_SECRET=your-super-secret-jwt-key-change-this
  JWT_EXPIRES_IN=7d
  DATABASE_PATH=database/app.db
  PORT=1221
  ```
- [ ] Обновить `backend/.env.example` с описаниями
- [ ] Убедиться, что `.env` в `.gitignore`

### Этап 2: Создание конфигурационного модуля
- [ ] Создать `backend/src/config/env.validation.ts`
- [ ] Реализовать валидацию обязательных переменных при старте
- [ ] Создать typed interface для конфигурации
- [ ] Бросать ошибку если обязательные переменные отсутствуют

### Этап 3: Обновление AuthModule
- [ ] Удалить fallback значение для JWT_SECRET
- [ ] Сделать JWT_SECRET обязательным
- [ ] Обновить signOptions для использования JWT_EXPIRES_IN из env

### Этап 4: Обновление Database конфигурации
- [ ] Вынести database path в переменную окружения
- [ ] Вынести synchronize в env (TYPEORM_SYNCHRONIZE)
- [ ] Вынести logging в env (TYPEORM_LOGGING)

### Этап 5: Обновление main.ts
- [ ] Вынести PORT в env variable
- [ ] Добавить CORS configuration через env (если нужно)

### Этап 6: Документация
- [ ] Обновить README с секцией Environment Variables
- [ ] Создать `backend/docs/environment-setup.md`
- [ ] Документировать все переменные окружения
- [ ] Добавить troubleshooting для missing variables

### Этап 7: Безопасность
- [ ] Убедиться что `.env` не коммитится в git
- [ ] Проверить что в `.env.example` нет реальных секретов
- [ ] Добавить генератор случайного JWT_SECRET (скрипт)

## Критерии приёмки
- ✅ Все секреты вынесены в `.env`
- ✅ Обязательные переменные валидируются при старте
- ✅ Fallback значения удалены
- ✅ `.env.example` полный и понятный
- ✅ Документация обновлена
