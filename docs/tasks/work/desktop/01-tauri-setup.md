# Задача DT-01: Настройка и интеграция Tauri Desktop

## Зависимости
- FE-04: Вынос API URL в переменные окружения (для правильной конфигурации)
- BE-05: Вынос секретов в переменные окружения (для безопасности)

## Описание проблемы
В проекте есть папка `src-tauri` с базовой конфигурацией, но desktop приложение не интегрировано с frontend и backend. Необходимо:
- Настроить Tauri для корректной работы с Vite
- Интегрировать frontend UI в desktop приложение
- Настроить коммуникацию с backend API
- Создать инсталлятор для Windows

## Цель задачи
Создать полнофункциональное desktop приложение на Tauri с интеграцией frontend и backend.

## Чек-лист выполнения

### Этап 1: Проверка и обновление Tauri
- [ ] Проверить версию Tauri в `src-tauri/Cargo.toml`
- [ ] Обновить до последней стабильной версии (если нужно)
- [ ] Проверить `src-tauri/tauri.conf.json` конфигурацию
- [ ] Обновить настройки window (размеры, title, resizable)

### Этап 2: Интеграция с Vite frontend
- [ ] В корневом `package.json` настроить скрипты:
  ```json
  "tauri:dev": "tauri dev",
  "tauri:build": "tauri build"
  ```
- [ ] Настроить `tauri.conf.json` для использования Vite dev server
- [ ] Проверить что `beforeDevCommand` и `beforeBuildCommand` корректны

### Этап 3: Настройка IPC (Inter-Process Communication)
- [ ] Создать `src-tauri/src/commands.rs` для Tauri команд
- [ ] Реализовать команды:
  - `get_app_version()` - получение версии приложения
  - `open_external(url: String)` - открытие URL в браузере
  - Другие специфичные desktop команды
- [ ] Обновить `src-tauri/src/main.rs` для регистрации команд

### Этап 4: Frontend integration
- [ ] Создать `frontend/src/utils/tauri.ts` - обёртка над Tauri API
- [ ] Определять Tauri environment:
  ```typescript
  const isTauri = window.__TAURI__ !== undefined;
  ```
- [ ] Адаптировать API URL для desktop (local backend или remote)

### Этап 5: Системная интеграция
- [ ] Настроить system tray (опционально)
- [ ] Настроить app menu (File, Edit, View, Help)
- [ ] Добавить иконки приложения (уже есть в `src-tauri/icons`)
- [ ] Настроить auto-update (опционально для MVP)

### Этап 6: Безопасность и разрешения
- [ ] Настроить CSP (Content Security Policy) в `tauri.conf.json`
- [ ] Настроить allowlist для API permissions
- [ ] Ограничить доступ к файловой системе (если нужно)

### Этап 7: Build конфигурация
- [ ] Настроить identifier в `tauri.conf.json`
- [ ] Настроить bundle для Windows (MSI installer)
- [ ] Проверить updater configuration (если используется)
- [ ] Добавить код подписи (code signing) для production

### Этап 8: Тестирование
- [ ] Создать тестовый build: `npm run tauri:build`
- [ ] Проверить работу инсталлятора на чистой системе
- [ ] Протестировать все функции в desktop приложении

### Этап 9: Документация
- [ ] Обновить корневой README.md с инструкциями по desktop
- [ ] Создать `docs/desktop-setup.md`
- [ ] Документировать процесс сборки и деплоя
- [ ] Добавить troubleshooting секцию
