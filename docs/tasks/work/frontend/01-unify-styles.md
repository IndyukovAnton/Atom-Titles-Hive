# Задача 01: Унификация подхода к стилизации

## Описание проблемы
В проекте одновременно используются:
- TailwindCSS v4 с конфигурацией
- Vanilla CSS файлы (Auth.css, Home.css, Modal.css, Profile.css, Settings.css, ThemeToggle.css)
- Инлайн-стили в компонентах
- shadcn/ui компоненты

Это создаёт:
- Непредсказуемость каскада стилей
- Дублирование кода
- Затруднённую поддержку
- Конфликты стилей

## Цель задачи
Выбрать единый подход к стилизации и мигрировать весь код на него.

## Чек-лист выполнения

### Этап 1: Анализ и выбор стратегии
- [ ] Провести аудит всех CSS-файлов и компонентов
- [ ] Определить стратегию: TailwindCSS + shadcn/ui ИЛИ CSS-модули
- [ ] Документировать решение в `docs/architecture/styling-decision.md`

### Этап 2: Подготовка конфигурации
- [ ] Обновить `tailwind.config.ts` (если выбран TailwindCSS)
- [ ] Настроить CSS-модули в `vite.config.ts` (если выбраны они)
- [ ] Создать файл с общими утилитами/миксинами

### Этап 3: Миграция компонентов
- [ ] Мигрировать `LoginPage.tsx`
- [ ] Мигрировать `RegisterPage.tsx`
- [ ] Мигрировать `ForgotPasswordPage.tsx`
- [ ] Мигрировать `HomePage.tsx`
- [ ] Мигрировать `ProfilePage.tsx`
- [ ] Мигрировать `SettingsPage.tsx`
- [ ] Мигрировать модальные окна (`AddMediaModal.tsx`, `CreateGroupModal.tsx`)
- [ ] Мигрировать UI-компоненты (`Button`, `ThemeToggle`, `ContextMenu`)

### Этап 4: Удаление устаревшего кода
- [ ] Удалить неиспользуемые CSS-файлы из `src/styles/`
- [ ] Удалить инлайн-стили из компонентов
- [ ] Очистить imports в компонентах

### Этап 5: Обновление документации
- [ ] Обновить README с инструкциями по стилизации
- [ ] Создать style guide в `docs/style-guide.md`
- [ ] Добавить примеры использования в документацию
