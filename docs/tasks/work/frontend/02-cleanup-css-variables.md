# Задача 02: Очистка и оптимизация CSS-переменных

## Описание проблемы
В файле `src/index.css` обнаружено:
- Дублирование переменных для тем (`:root` и `[data-theme="dark"]` + `.dark`)
- Смешение устаревших переменных (`--bg-primary`, `--text-primary`) с современными oklch-токенами
- Неиспользуемые CSS-переменные
- Несогласованность механизмов переключения тем

## Цель задачи
Привести CSS-переменные к единому стандарту, удалить дублирование и обеспечить чёткий механизм переключения тем.

## Чек-лист выполнения

### Этап 1: Аудит переменных
- [ ] Создать список всех CSS-переменных в проекте
- [ ] Определить используемые и неиспользуемые переменные
- [ ] Проверить, какой механизм темы используется (`[data-theme]` или `.dark`)
- [ ] Найти все места использования переменных в компонентах

### Этап 2: Выбор стандарта цветов
- [ ] Определить: использовать oklch ИЛИ rgba/hsl
- [ ] Создать единую цветовую палитру (light/dark)
- [ ] Документировать выбранный подход в `docs/architecture/color-system.md`

### Этап 3: Рефакторинг index.css
- [ ] Удалить дублированные переменные для темной темы
- [ ] Выбрать один механизм: `[data-theme="dark"]` ИЛИ `.dark`
- [ ] Привести все переменные к единому формату
- [ ] Удалить неиспользуемые переменные (`--btn-primary-bg-hover`, `--icon-color` и т.д. если не используются)
- [ ] Группировать переменные логически (цвета, отступы, радиусы и т.д.)
- [ ] Добавить комментарии к группам переменных

### Этап 4: Обновление ThemeContext
- [ ] Проверить `src/contexts/ThemeContext.tsx` на соответствие с выбранным механизмом
- [ ] Убедиться, что переключение темы работает только через один механизм
- [ ] Удалить устаревшие обработчики смены темы

### Этап 5: Обновление компонентов
- [ ] Найти все использования устаревших переменных в компонентах
- [ ] Заменить на новые стандартизированные переменные
- [ ] Проверить shadcn/ui компоненты на использование правильных токенов

### Этап 6: Оптимизация @theme inline
- [ ] Проверить необходимость всех radius-переменных
- [ ] Убедиться, что все sidebar-переменные используются
- [ ] Удалить неиспользуемые chart-переменные (если не нужны)

## Критерии приёмки
- ✅ Отсутствует дублирование CSS-переменных
- ✅ Используется единый механизм переключения тем
- ✅ Все переменные документированы
- ✅ Код чистый и читаемый
- ✅ Размер index.css уменьшен минимум на 20%

## Тестирование

### Визуальная проверка тем
- [ ] Светлая тема отображается корректно на всех страницах
- [ ] Темная тема отображается корректно на всех страницах
- [ ] Переключение между темами происходит плавно
- [ ] Все цвета соответствуют дизайн-системе
- [ ] Контрастность текста достаточна (WCAG AA)

### Проверка компонентов
- [ ] Кнопки имеют правильные цвета в обеих темах
- [ ] Формы и инпуты корректно окрашены
- [ ] Модальные окна используют правильные цвета фона
- [ ] Границы элементов видны и согласованы
- [ ] Тени элементов адаптируются к темам

### Функциональная проверка
- [ ] Переключатель темы работает
- [ ] Выбранная тема сохраняется после перезагрузки страницы
- [ ] При первом посещении применяется системная тема (если реализовано)

### Проверка кода
- [ ] ESLint не выдаёт предупреждений
- [ ] TypeScript компилируется без ошибок
- [ ] CSS валиден (можно проверить через stylelint)
- [ ] Нет неиспользуемых переменных
- [ ] `npm run build` проходит успешно

### Проверка производительности
- [ ] Размер CSS bundle уменьшился
- [ ] Скорость переключения темы < 100ms
- [ ] Нет мерцания при смене темы
