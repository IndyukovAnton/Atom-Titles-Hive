# Задача 04: Вынос API URL в переменные окружения

## Описание проблемы
В файле `src/api/client.ts` захардкожен URL API:
```typescript
const API_URL = 'http://localhost:1221';
```

Это создаёт проблемы:
- Невозможно переключаться между dev/staging/production окружениями
- Нельзя запустить на другом порту без изменения кода
- Сложно настраивать CI/CD
- Нарушает best practices (конфигурация должна быть вне кода)

## Цель задачи
Вынести все конфигурационные параметры в переменные окружения и создать систему управления конфигурацией.

## Чек-лист выполнения

### Этап 1: Создание .env файлов
- [ ] Создать `.env.local` для локальной разработки
  - `VITE_API_URL=http://localhost:1221`
  - `VITE_API_TIMEOUT=30000`
  - `VITE_APP_NAME=Web Titles Tracker`
- [ ] Создать `.env.development` для dev окружения
- [ ] Создать `.env.production` для production окружения
- [ ] Создать `.env.example` с примерами всех переменных
- [ ] Добавить `.env*.local` в `.gitignore`

### Этап 2: Создание конфигурационного модуля
- [ ] Создать `src/config/env.ts` для валидации переменных
  ```typescript
  export const config = {
    apiUrl: import.meta.env.VITE_API_URL,
    apiTimeout: Number(import.meta.env.VITE_API_TIMEOUT),
    // ...
  };
  ```
- [ ] Добавить проверку обязательных переменных при старте
- [ ] Добавить TypeScript типы для конфигурации
- [ ] Создать `src/config/index.ts` barrel export

### Этап 3: Обновление API клиента
- [ ] Заменить захардкоженный `API_URL` на `config.apiUrl` в `src/api/client.ts`
- [ ] Добавить использование `config.apiTimeout` для axios
- [ ] Проверить все API модули на наличие других захардкоженных URL

### Этап 4: Обновление Vite конфигурации
- [ ] Проверить `vite.config.ts` на поддержку переменных окружения
- [ ] Убедиться, что `VITE_*` переменные доступны в клиенте
- [ ] Настроить proxy для dev-сервера (если нужно)
  ```typescript
  server: {
    proxy: {
      '/api': {
        target: process.env.VITE_API_URL,
        changeOrigin: true,
      }
    }
  }
  ```

### Этап 5: Обновление скриптов package.json
- [ ] Обновить команды для использования разных .env файлов
  ```json
  "dev": "vite --mode development",
  "build:prod": "vite build --mode production",
  "build:staging": "vite build --mode staging"
  ```
- [ ] Добавить скрипты для проверки конфигурации

### Этап 6: Документация
- [ ] Обновить README.md с инструкциями по настройке .env
- [ ] Создать `docs/environment-setup.md` с подробным описанием переменных
- [ ] Добавить troubleshooting секцию для частых проблем с конфигурацией
- [ ] Документировать процесс деплоя для разных окружений
