# Задача 12: Система уведомлений

## Описание
Централизованная система уведомлений для информирования пользователя о событиях.

## Цель задачи
Повысить интерактивность и информированность пользователя с помощью ненавязчивых уведомлений.

## Чек-лист выполнения

### Этап 1: UI Уведомлений
- [ ] **Центр уведомлений**: Отдельный раздел или выпадающая панель "Уведомления".
- [ ] **История**: Список прошлых уведомлений.
- [ ] **Управление**: Возможность пометить как прочитанное или удалить.

### Этап 2: Типы уведомлений
- [ ] **Категории**:
  - [ ] Системные.
  - [ ] Рекомендации.
  - [ ] Обновления приложения.
- [ ] **Toasts**: Всплывающие уведомления при активных действиях (Success/Error).

## Критерии приёмки
- ✅ Уведомления приходят в реальном времени (или при действии).
- ✅ История сохраняется.
- ✅ Можно очистить список.

## Технические требования
- WebSocket (опционально для server-push) или Polling.
- Local Storage для неподтвержденных уведомлений (если оффлайн).
