# Задача 14: Раздел обновлений и безопасность

## Описание
Информирование пользователей об изменениях в проекте и обеспечение безопасности данных.

## Цель задачи
Прозрачность разработки для пользователя и защита чувствительных данных.

## Чек-лист выполнения

### Этап 1: Раздел "Обновления"
- [ ] **Страница**: Создать страницу Changelog.
- [ ] **Данные**: Автоматическая генерация или ручное ведение истории изменений (Версия, Дата, Описание).
- [ ] **Уведомление**: Индикатор "Что нового" при обновлении фронтенда.

### Этап 2: Безопасность
- [ ] **Политика**: Обновить текст Политики конфиденциальности.
- [ ] **API Keys**: Реализовать безопасное хранение ключей пользователя (шифрование на клиенте или проксирование без сохранения на бэкенде в открытом виде).
- [ ] **Приватность**: Настройки приватности для раздела рекомендаций (какие данные отправлять AI).

## Критерии приёмки
- ✅ Пользователь видит список изменений.
- ✅ API ключи не утекают в сеть (кроме запросов к провайдеру).

## Технические требования
- Интеграция с Git (опционально, на этапе сборки генерировать JSON с коммитами).
